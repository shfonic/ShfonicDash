# F1 2025 UDP Telemetry Reference

Applies to: F1 2025 (EA Sports / Codemasters). The F1 25: 2026 Season Pack broadcasts `m_packetFormat = 2026` and extends `CarStatusData` with one extra field; everything else is identical to the 2024/2025 layout.

Official specification (F1 24 base):  
https://answers.ea.com/t5/General-Discussion/F1-24-UDP-Specification/m-p/13745808

---

## In-Game Setup

Settings → Telemetry Settings → UDP Telemetry: On  
IP Address: Pi's LAN address  
Port: **20777**  
Format: 2024 (or 2026 for the Season Pack)  
Send Rate: 60 Hz recommended

---

| Offset | Type | Field | Notes |
|--------|------|-------|-------|
| 0 | U16 | `m_packetFormat` | 2024 (F1 25) or 2026 (Season Pack) |
| 2 | U8 | `m_gameYear` | e.g. 25 |
| 3 | U8 | `m_gameMajorVersion` | |
| 4 | U8 | `m_gameMinorVersion` | |
| 5 | U8 | `m_packetVersion` | |
| 6 | U8 | `m_packetId` | See packet IDs below |
| 7 | U8×8 | `m_sessionUID` | Unique session identifier |
| 15 | F32 | `m_sessionTime` | Seconds since session start |
| 19 | U32 | `m_frameIdentifier` | |
| 23 | U32 | `m_overallFrameIdentifier` | |
| 27 | U8 | `m_playerCarIndex` | Index of player's car in multi-car arrays |
| 28 | U8 | `m_secondaryPlayerCarIndex` | 255 = no secondary player |

## Packet IDs

| ID | Name | Used | Notes |
|----|------|------|-------|

| 1 | Session | ✓ | Session type, formula class, weather, safety car, track info |

| 3 | Event | — | One-shot events: fastest lap, retirement, DRS enabled/disabled, etc. |
| 4 | Participants | ✓ | Driver names, team names, nationalities for all cars |
| 5 | Car Setups | — | Wing angles, suspension, tyre pressures |
| 6 | Car Telemetry | ✓ | Speed, throttle, brake, gear, RPM, DRS, tyre temps, pressures |

| 9 | Lobby Info | — | Multiplayer lobby data |
| 10 | Car Damage | — | Tyre wear, damage percentages per component |

| 12 | Tyre Sets | — | All tyre sets available/used this session |
| 13 | Motion Ex | — | Extended motion data for player car only |
| 16 | Car Telemetry 2 | ✓ | F1 2026 DLC: Active Aero mode, Boost (MOR) per car |

| Value | Session |
|-------|---------|

## Packet 2 — Lap Data

Per-car struct: **57 bytes**. Array starts at byte 29.

| Offset | Type | Field | Used | Notes |
|--------|------|-------|------|-------|
| 0 | U32 | `m_lastLapTimeInMS` | ✓ | 0 if no completed lap |
| 4 | U32 | `m_currentLapTimeInMS` | ✓ | Running lap time |
| 8 | U16 | `m_sector1TimeInMS` | ✓ | ms part of S1 time (max 59 999) |
| 10 | U8 | `m_sector1TimeMinutes` | ✓ | Minutes part of S1 |
| 11 | U16 | `m_sector2TimeInMS` | ✓ | ms part of cumulative S2 time |
| 13 | U8 | `m_sector2TimeMinutes` | ✓ | Minutes part of cumulative S2 |
| 14 | U16 | `m_deltaToCarInFrontInMS` | — | Gap to car ahead |
| 17 | U16 | `m_deltaToRaceLeaderInMS` | — | Gap to leader |
| 20 | F32 | `m_lapDistance` | ✓ | Distance into current lap (m) — used for delta/profile recording and pit detection |
| 24 | F32 | `m_totalDistance` | — | Total session distance (m) |
| 28 | F32 | `m_safetyCarDelta` | — | Delta to safety car |
| 32 | U8 | `m_carPosition` | ✓ | Race position (1-based) |
| 33 | U8 | `m_currentLapNum` | ✓ | Current lap number |
| 34 | U8 | `m_pitStatus` | ✓ | 0=none, 1=pitting, 2=in pit area — suppresses delta and profile recording |
| 35 | U8 | `m_numPitStops` | — | Pit stops taken |
| 36 | U8 | `m_sector` | ✓ | Current sector (0, 1, 2) |
| 37 | U8 | `m_currentLapInvalid` | ✓ | 1 = lap invalid (track limits) |
| 38 | U8 | `m_penalties` | — | Seconds of time penalty |
| 39 | U8 | `m_totalWarnings` | — | |
| 40 | U8 | `m_cornerCuttingWarnings` | — | |
| 41 | U8 | `m_numUnservedDriveThroughPens` | — | |
| 42 | U8 | `m_numUnservedStopGoPens` | — | |
| 43 | U8 | `m_gridPosition` | — | Starting grid position |
| 44 | U8 | `m_driverStatus` | — | 0=garage, 1=flying lap, 2=in lap, 3=out lap, 4=on track |
| 45 | U8 | `m_resultStatus` | — | 0=invalid, 1=inactive, 2=active, 3=finished, 4=DNF, etc. |
| 46 | U8 | `m_pitLaneTimerActive` | — | |
| 47 | U16 | `m_pitLaneTimeInLaneInMS` | — | |
| 49 | U16 | `m_pitStopTimerInMS` | — | |
| 51 | U8 | `m_pitStopShouldServePen` | — | |
| 52–56 | — | (reserved) | — | |

**Note on sector times:** `m_sector1TimeInMS` and `m_sector2TimeInMS` are the ms portion only; combine with the corresponding `Minutes` byte as `minutes × 60 000 + ms`. `m_sector2Time` is the cumulative time to the end of S2, not standalone S2. S3 = `lastLap − sector2`.
